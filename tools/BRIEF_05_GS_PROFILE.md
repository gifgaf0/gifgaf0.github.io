# Brief 05 — Gram-Schmidt Profile of the SLWE Primal Lattice (OP-F)

> *T1 = measured fact (script reproducible from `tools/gs_profile.py`).
> T2 = structural argument used to interpret the measurement.
> No prediction or extrapolation is labelled T1.*

## Headline (T1)

**The Singer-orbit construction of the SLWE public matrix `A`
collapses the rank of the flattened `A_F` matrix by roughly 6.7×
at `k = 32`.** Concretely, at `(k = 32, p = 8191, n = k·d = 512)`:

| Matrix | F_p-rank | Z-column-period | GS-zero residuals (out of 513) |
|---|---:|---:|---:|
| **Singer-`A`**  | **76**  | **112** | **436** |
| **Random-`A`**  | 512     | none    | 0       |

Both bases were built identically: rows are the 512 rows of `A_F`
plus one Kannan-style q-tag row, giving a `513 × 513` integer
basis. Random `A_F` is generated with i.i.d. uniform-in-`[0, p)`
sedenion components; Singer `A_F` is built from PSL(2,7) Z₇
Singer-cycle orbits per the construction in `tools/sqt_slwe.py`.

The brief's flag threshold is "max log₂-difference > 0.5". The
observed signal is qualitatively different: **436 of the 513 GS
residuals collapse to zero in Singer's basis but are all nonzero
in Random's**. The flag fires by a wide margin.

## §1. Setup (T1)

- **Lattice basis.** `(n+1) × (n+1)` with `n = k · d = 32 · 16 =
  512`. Rows 0..511 are the rows of the flattened public matrix
  `A_F`; row 512 is the Kannan tag `(p, 0, …, 0, 1)`. The
  flattening replaces each sedenion entry `A[i, j] ∈ S_p` with its
  16×16 left-multiplication matrix on `S_p` (same convention as
  `tools/sqt_slwe.py::rank_flat`).
- **Backend.** `fpylll.GSO` with `float_type='mpfr'`, precision
  200 bits. Numpy fallback wired but not invoked here — fpylll
  installed cleanly via `pip install cysignals fpylll`.
- **Reproducibility.** `python3 tools/gs_profile.py --k 32 --p 8191`.
  Wall clock on this host: 24 s total (0.4 s flatten, 12 s rank
  computation mod p, 12 s GS).

## §2. Why rank deficit, structurally (T2)

The Singer cycle σ is the order-7 permutation `e_i ↦ e_{(i mod 7)+1}`
on the imaginary basis indices 1..7 of S_p. The Singer-orbit
construction sets `A_sed[i][j] = σ^j(seed_i)`. Because σ has order
7, `σ^{j+7} = σ^j` and therefore

```
A_sed[:, j] = A_sed[:, j + 7]      (column equality at distance 7
                                    in the sedenion-level matrix)
```

After flattening each sedenion column to a 16-block of F_p
columns, this becomes column equality at distance 7 · 16 = 112
in the F_p-level matrix `A_F`:

```
A_F[:, j] = A_F[:, j + 112]        (verified directly,
                                     T1 measurement)
```

For `k = 32`, `A_F` has 512 columns but only 112 distinct ones
(every column j ≥ 112 is a literal copy of column j mod 112).
Hence column rank ≤ 112. By rank duality, row rank ≤ 112. So
the lattice generated by `A_F`'s rows has dimension at most
112, not 512 as the matrix shape would suggest.

The empirical Z-rank we observe via GS — 76 — is even smaller
than 112, because the seeds chosen for each row are not
independent over F_p (rand_nonzd's filter and the algebra's
zero-divisor structure cause additional collinearities). The
**76** is a tight bound for this particular sample; the **112**
is the structural ceiling that holds for every Singer-orbit
realisation regardless of seed choice.


`< −100` denotes residuals below the threshold of 10⁻¹⁰ (well
into the regime where mpfr-200 returns small negative values
that we clip to zero). The full `log₂ ‖b*ᵢ‖` array for both
profiles is in `tools/gs_profile_results.json`.

### Aggregate statistics

| | Singer | Random |
|---|---:|---:|
| Slope of log₂ profile (least squares over nonzero indices) | −0.039 | −0.0044 |
| RMS deviation from line | 96.9 bits | 0.41 bits |
| Max abs deviation from line | 113 bits | 3.76 bits |
| Two-sigma deviations from own GSA line | 436 indices | 0 indices |

The Singer slope and RMS are dominated by the cliff at index 76.
Restricted to indices 0..75 (where Singer is nonzero), the
profile slopes match closely:

- Indices 0..50: Singer ≈ Random (max |Δ| < 0.5 bits)
- Indices 51..75: Singer drops faster than Random (Δ reaches
  −5.9 bits at index 75)
- Index ≥ 76: Singer = 0; Random ≈ 15.6 bits

This is the smoking-gun shape: Singer's GS norms behave like a
random matrix's for the first ~50 rows, then deviate sharply over
the last ~25 nonzero rows (rank starvation as the basis fills),
then collapse to zero entirely.

### Signer-vs-Random comparison (T1)

- **`max |Δ log₂| on indices where both nonzero` = 5.901
  bits at index 75.** Threshold for the brief's flag was 0.5;
  the observed signal is ~12× over threshold even *before*
  considering the rank cliff.
- **`rank_collapse_index_count = 436`** — i.e., 436 indices where
  Singer's GS norm is zero and Random's is nonzero. By any
  reasonable interpretation of the brief's flag, this fires.

## §4. What this means for security (T2)

**Direct cryptanalytic implication.** A primal-attack lattice
built from one Module-SLWE sample with this Singer A has rank
≤ 113 instead of 513. An attacker who recognises the structure
(trivially: any row test of `A_F[:, j] == A_F[:, j+112]`) can:

1. Reduce the SLWE problem to its 76-dim quotient where the
   structure is fully exposed.
2. Attack the quotient lattice directly. At dimension 76, BKZ
   block sizes that would be infeasible at 512 are routine.
3. Lift the 76-dim solution back to the original 512-dim
   secret using the Singer cycle's action.

The rank deficit is not a "potential" weakness needing further
analysis. It is an algebraic identity: columns at distance 112
are literal copies. (T2.)

**Effect on the Brief 04 closure verdict.** Brief 04 concluded
that the Module-SLWE construction's security "rests entirely on
Module-LWE; the non-associative algebra provides diversification
against ideal-lattice folding attacks but contributes no
independent multiplicative hardness." This brief shows the
diversification claim *also fails at k = 32*: the Singer-orbit
public matrix structure is itself an ideal-lattice-style folding
in disguise, and the rank cliff is the folding becoming visible.
At k = 32 there is no usable lattice security either. (T2.)

**What would fix it.** Either:

- Use `k ≤ 7` (so the Singer cycle never repeats inside one
  row; column-rank bound becomes ≥ k·d, and at k = 7, A_F is
  full rank 7·16 = 112). At k = 7 the SLWE secret dimension is
  7 · 16 = 112 — too small for cryptographic use, but
  structurally consistent.
- Or choose the Singer orbit length to match the module rank
  (cycle of order ≥ k), which requires changing the prime away
  from mod-455 toward primes admitting larger Singer cycles.
- Or drop the Singer-orbit structure on `A` entirely. Use a
  uniformly random A. The non-associative algebra's
  diversification claim would then need to be re-justified
  without leaning on PSL(2, 7).

I have not implemented any of these; they require redesign
choices outside this brief's scope.

**(a)** Singer column-period = 112 at k = 32 (i.e. columns at
distance 112 are literal integer copies). **(b)** Singer F_p-rank
≤ 112 at k = 32. **(c)** Random `A_F` is F_p full rank at k = 32.

Documentation-only (in this report and `gs_profile_results.json`):
the full 513-entry GS-norm profile for Singer and Random; the
slope / RMS-deviation / max-deviation aggregates; the
indexwise comparison and rank-cliff location at index 76; the
empirical Singer F_p-rank value 76 (which is sample-specific,
unlike the structural ceiling 112). The cryptanalytic
interpretation (lattice attack reducing to dim 76 / 112) is also
documentation-only — no exploit script lives in this branch.
