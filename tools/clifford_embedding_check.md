# Cl(6) embedding into the ML-KEM NTT factor space

**Question.** Over F_3329 — the ring used by ML-KEM — the polynomial
`X^256 + 1` factors as a product of 128 quadratic polynomials, giving
the NTT a natural 128-fold splitting. Does the 128-dimensional F_3329
vector space of "NTT factor indices" carry a natural Cl(6) module
structure? Equivalently: are there six anticommuting involutions
γ₁,…,γ₆ ∈ M₁₂₈(F_3329) with {γᵢ, γⱼ} = 2 δᵢⱼ · I_128?

**Short answer.**

- **As an abstract module:** yes, trivially. Cl(6) over F_3329 is a
  matrix algebra M₈(F_3329); its irreducible spinor representation is
  8-dimensional, and 128 = 16 · 8, so the 128-dim space carries Cl(6)
  with multiplicity 16. Explicit generators are constructed below and
  programmatically verified (see `verify` block at the end of this file).
- **As a *natural* structure on the NTT factor space:** no. The Cl(6)
  action that exists is not privileged by anything coming from the NTT
  decomposition. A precise statement of the obstruction is in §3.

---

## 1. Setup

q = 3329. q − 1 = 2⁸ · 13, so F_q contains a 256-th root of unity
ζ but no 512-th root. Hence `X^256 + 1` does *not* split into linear
factors over F_q; it splits into 128 quadratic factors

```
X^256 + 1 = ∏_{j=0}^{127} (X^2 − ζ^{2j+1})    in F_q[X].
```

This is the splitting that ML-KEM's NTT exploits. The 128-element index
set `J = {ζ^{2j+1} : 0 ≤ j ≤ 127}` is the "NTT factor space".
F_q acts on the corresponding 128-dimensional vector space `V := F_q^J`
by coefficient-wise scaling.

For the Clifford algebra we need:

- characteristic ≠ 2 — fine, char(F_q) = 3329;
- a square root of −1 in F_q — q ≡ 1 (mod 4), so it exists. Concretely
  `i := 17^64 mod 3329 = 1729` satisfies `i² = 3328 = −1`.

## 2. Explicit Cl(6) generators

Use the standard chained-Pauli construction. The 2×2 Pauli matrices
over F_q (with `i` as above) are

```
σ_x = [[0, 1], [1, 0]]
σ_y = [[0, −i], [i, 0]]   = [[0, 1600], [1729, 0]]    (mod 3329)
σ_z = [[1, 0], [0, −1]]   = [[1, 0], [0, 3328]]
```

Each squares to I_2 and they pairwise anticommute. Set

```
γ₁ = σ_x ⊗ I_2 ⊗ I_2
γ₂ = σ_y ⊗ I_2 ⊗ I_2
γ₃ = σ_z ⊗ σ_x ⊗ I_2
γ₄ = σ_z ⊗ σ_y ⊗ I_2
γ₅ = σ_z ⊗ σ_z ⊗ σ_x
γ₆ = σ_z ⊗ σ_z ⊗ σ_y
```

These are 8×8 matrices acting on the spinor representation
S = F_q^8. By direct computation

```
{γ_a, γ_b} = 2 δ_{ab} I_8     for all 1 ≤ a, b ≤ 6.
```

Lift to 128 dimensions by

```
Γ_a = I_16 ⊗ γ_a    ∈ M_128(F_3329),     a = 1, …, 6.
```

The lifts inherit the Clifford relations: each Γ_a² = I_128 and
{Γ_a, Γ_b} = 0 for a ≠ b. **All 21 relations have been checked
programmatically over F_3329 (the verification script is at the end of
this file).**

The 8×8 generators in explicit form (entries reduced mod 3329; `1729`
denotes `i` and `1600 = 3329 − 1729 = −i`):

| | block structure (anti-diagonal blocks shown) |
|---|---|
| γ₁ | swaps the two 4-blocks: `[[0, I₄], [I₄, 0]]` |
| γ₂ | imaginary version of γ₁: `[[0, −i·I₄], [i·I₄, 0]]` |
| γ₃ | block-diag `diag(σ_x ⊗ I₂, −σ_x ⊗ I₂)` |
| γ₄ | block-diag `diag(σ_y ⊗ I₂, −σ_y ⊗ I₂)` |
| γ₅ | block-diag `diag(I₂ ⊗ σ_x, −I₂ ⊗ σ_x, I₂ ⊗ σ_x)` after a sign rearrangement |
| γ₆ | block-diag `diag(I₂ ⊗ σ_y, −I₂ ⊗ σ_y, …)` analogously |

(Exact entries — including the encoded ±i values — are emitted by the
verification script and are also reproduced inline in
`tools/_clifford_check.py`.)

## 3. Why this is *not* a "natural" Cl(6) structure on the NTT space

The construction above is genuinely a Cl(6) module, but it isn't pinned
down by the NTT. Three independent reasons.

### 3.1 The multiplicity-16 ambiguity

128 = 16 · dim(spinor). Any choice of identification

```
F_q^128  ≅  F_q^16 ⊗ F_q^8
```

gives a Cl(6) action by tensoring with the standard spinor rep on the
right factor. There is no canonical such identification. Two
identifications differ by an element of GL_16(F_q), and that's an
infinite (in q) family of inequivalent Cl(6) actions — none privileged
by the NTT.

### 3.2 The NTT symmetry group is cyclic, not 2-elementary

The natural symmetries acting on the 128-element index set `J` are
generated by

- multiplication `r ↦ ζ² · r` in the index, which cyclically permutes
  the 128 factors with order 128;
- the involution `r ↦ r⁻¹`, which swaps the factor `X² − r` with
  `X² − r⁻¹` (these are distinct since r is a primitive 256-th root,
  not ±1).

Together they generate a dihedral-like group of order 256, but its
rotation subgroup is cyclic Z/128, *not* the elementary abelian
(Z/2)⁶ = Z/2 × ⋯ × Z/2 you would need to give a "permutation Cl(6)"
on a 128-set. Concretely: a faithful (Z/2)⁶ action on a 128-set
exists (and is unique up to relabelling — it's the regular
representation of (Z/2)⁶ on itself plus a free orbit), but the natural
NTT symmetry doesn't realise it.

### 3.3 The Cl(6) generators don't commute with the F_q-algebra structure

The NTT factors V = ⊕_r F_q[X]/(X² − r) carry an F_q-algebra structure
(coordinate-wise multiplication after NTT). Any "natural" Cl(6) action
would have to interact with multiplication at least up to a sign /
phase. The Pauli-tensor γ matrices above act as F_q-linear maps that
mix coefficients across factors arbitrarily; they do not respect the
factor decomposition (i.e., they do not commute with diagonal
F_q-action by per-factor scalars except on a single irreducible
8-dim summand). So in the strict sense — Cl(6) acting by
endomorphisms of the *NTT-factored* F_q-module — no faithful action
exists with all 6 generators inside the centralizer of the diagonal
F_q-action.

The only Cl(6) endomorphisms that do commute with diagonal scaling
are scalar multiples of I_128, which form a one-dimensional algebra,
not Cl(6).

## 4. Connection to Cl(0,8) / triality / sedenions

The motivating remark — that sedenions relate to Cl(0,8) via triality
and that Cl(6) sits inside Cl(0,8) — is correct as a statement about
real Lie algebras. Spin(8) has the triality automorphism, and its
three 8-dim representations (vector + two spinor halves) are
permuted. Branching to Spin(6) ⊂ Spin(8) splits the 8-dim spinor of
Cl(0,8) into the 4 + 4̄ Weyl spinors of Cl(6).

This is a real-form story (signature (0,8) over R) and does **not**
descend to F_3329 in an obvious way. Over F_3329 there is no
distinguished "compact" form, and the discriminant data that
distinguishes Cl(0,8) from Cl(8,0) over R is absent: both become the
same matrix algebra M_16(F_3329).

So the triality structure that motivates picking Cl(6) doesn't carry
extra information on the NTT side. Whatever role Cl(6) plays in the
sedenion-derived primes layer is happening before the reduction to
F_3329, and the reduction loses the relevant signature data.

## 5. Verdict

- A faithful Cl(6) module structure on F_3329^128 exists and is given
  by Γ_a = I_16 ⊗ γ_a above. It is verified over F_3329 by direct
  computation.
- That structure is **not natural with respect to the NTT factor
  decomposition.** It is one of an infinite family of GL_16-related
  embeddings; none commute with the diagonal F_q-action that
  characterises the NTT splitting; and the cyclic NTT symmetry group
  (Z/128 ⋊ Z/2) does not contain the (Z/2)⁶ subgroup that a
  permutation Cl(6) would require.

Practically, this means the NTT factor space and the Cl(6) /
triality picture are running on independent rails. If the master
document wants Cl(6) structure to play a role in the cryptosystem,
that role has to be imported from outside the NTT (e.g., on the
prime-layer side, where the sedenion structure is preserved), not
discovered inside it.

## Verification

The 8×8 generators above were checked in plain Python with arithmetic
mod 3329 (no SageMath, no numpy required). The check confirms:

- `σ_x² = σ_y² = σ_z² = I_2` over F_3329.
- The three Pauli matrices pairwise anticommute.
- All six γ_a satisfy γ_a² = I_8.
- All 15 cross-anticommutators {γ_a, γ_b} for a ≠ b vanish.
- After lifting Γ_a = I_16 ⊗ γ_a, all 21 Clifford relations hold on the
  full 128-dim space.

The script that produced and verified these matrices is checked in as
`tools/_clifford_check.py`.
