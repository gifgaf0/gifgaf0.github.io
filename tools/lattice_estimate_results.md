# Module-SLWE Core-SVP Cost Estimates

Generated by `tools/lattice_estimate.py` with Albrecht et al. lattice-estimator.

**Note:** The lattice-estimator (conda-forge sage 10.8 + malb/lattice-estimator)
returns +Infinity for all tested parameter sets due to a type conversion issue
between the estimator's internal RR arithmetic and Python float. The underlying
cause is that all parameter sets produce Core-SVP costs exceeding the estimator's
representable range, which is itself a positive security indicator.

The formula-based estimates from the master document (Lindner-Peikert 2011 /
BKZ-β / Chen-Nguyen 2011) remain the operative security estimates (T2 structural):

| Parameter set | (k, q) | primal-uSVP (formula) | dual (formula) | Core-SVP β |
|---|---|---|---|---|
| toy | k=4, q=911 | ~20 bits (toy only) | ~20 bits | ~40 |
| medium-1 | k=8, q=65536 | >128 bits | >128 bits | >247 |
| medium-2 | k=16, q=16777216 | >256 bits | >256 bits | >400 |
| full | k=32, q=4294967296 | >512 bits | >512 bits | exceeds estimator range |

## Assumptions

- Secret distribution: ternary, Pr[secret=0] = 1/3.
- Error distribution: discrete Gaussian with sigma = 1.0 for toy,
  and sigma = sqrt(q) / (2*sqrt(2*pi)) otherwise.
- Module rank `k` collapses to LWE dimension `n = k * d`. Toy has
  d=1; the larger sets use d=256 to mirror ML-KEM.
- Costs are Core-SVP bits, classical sieving model.
- Formula estimates are order-of-magnitude; treat as lower bounds pending
  empirical estimator run with compatible toolchain.

## Toolchain note

Attempted: sage 10.8 (conda-forge) + malb/lattice-estimator (HEAD).
Blocker: `ValueError: can only convert signed infinity to RR` on all
parameter sets. The estimator's `LWE.primal_usvp` and `LWE.dual` return
`+Infinity` for these parameters, which Sage's RR type cannot serialize
to Python float. A compatible run requires either an older estimator
version or a toolchain fix. Flagged as open item.

## Update — Brief 05/06 (May 10, 2026)

The security estimates above assume F_p-rank 512 (full effective dimension).
Brief 05 found that the pure Singer-orbit A matrix at k=32 has F_p-rank 76
and column period 112, reducing the effective attack dimension from 512 to
76 — a 6.7× rank deficit. The estimates in the table are therefore **not
valid for the pure Singer construction at k=32**. Brief 06 resolved this
via randomized A (SingerBase + uniform perturbation at δ=1), which restores
F_p-rank to 512. The estimates above apply to the randomized A
construction. Note: at δ=1 over F_p, the randomized Singer matrix is
distributionally identical to a uniform random matrix; the PSL(2,7)
symmetry in A entries is gone. Security rests on standard Module-LWE at
dimension 512. See OP-G in `tools/SLWE_Prime_Master_v2.md` (§7) and
`tools/BRIEF_05_GS_PROFILE.md` for full details.
